## Required Tools

- Glob
- Read
- Write

## Input Parsing

Parse `$ARGUMENTS` for:
- `--output <path>`: optional — file path to write the README. Defaults to `README.md` in the current working directory.

## Step 1 — Discover Project Files

Use Glob to find all skill definition files:
```
pattern: ".claude/commands/*/SKILL.md"
```

Also locate these files if present (read whichever exist):
- `CLAUDE.md` — project rules and context
- `SPEC.md` — project specification and data flow

Read each discovered file with the Read tool and extract:
- From `CLAUDE.md`: project description, rules, configuration
- From `SPEC.md`: project overview, data flow diagram, command table, phase descriptions
- From each `SKILL.md`: `name`, `description`, `argument-hint`, and the full list under `## Required Tools`

## Step 2 — Scan Directory Structure

Use Glob with pattern `**/*.{md,txt,json}` (excluding `.claude/` internals and `node_modules`) to understand the project layout. Note any key files at the root level.

## Step 3 — Generate README Content

Using the collected information, compose a README.md with the following sections in order:

### Section 1 — Title and Overview

```
# <Project Name>

<2–3 sentence description of what this project does, its core pipeline, and its purpose.>
```

Derive the project name from CLAUDE.md or SPEC.md content.

### Section 2 — Data Flow

Include the Mermaid diagram from SPEC.md verbatim if present. If not present, derive a diagram from the skills discovered:

```
## Data Flow

Dynamically generate a Mermaid data flow diagram (`graph TD`) illustrating how the User, Claude Code (GSD framework), Notion MCP, Notion API, and local filesystem interact, based strictly on your understanding of the discovered skills and architecture.

### Section 3 — Prerequisites

List all prerequisites inferred from the skills and project context:

```
## Prerequisites

- [Claude Code CLI](https://claude.ai/claude-code) — the primary interface and AI orchestrator
- Notion MCP Server — provides access to the Notion API; must be configured in `.claude/settings.json`
- A Notion integration token with read/write access to the target workspace
- Node.js (for running Claude Code)
```

### Section 4 — Setup

```
## Setup

1. Clone this repository.
2. Open the project in a terminal and run `claude` to start Claude Code.
3. Ensure the Notion MCP server is configured in `.claude/settings.json` with your integration token.
4. Verify the integration has access to the Notion pages you intend to use.
```
